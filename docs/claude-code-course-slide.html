Section 02 · 四種操作模式

四種操作模式

切換：Shift + Tab
Normal mode問答模式，若要執行指令、寫檔操作都會先問你同不同意不確定它會做什麼時
Plan mode先輸出完整計畫讓你審核，確認後才動手大改動前

Auto mode全自動，什麼都不問跑確定的重複任務

Section 03 · 特殊符號

特殊符號 / ! @

/
呼叫 slash commands
> /exit　　離開 CLI
> /clear　　清空 session，開新對話
> /skills　列出所有可用 Skills
!
直接執行 shell 指令
> !ls -la
> !git status && git diff
@
把指定檔案加進對話
> @note.txt 幫我重整筆記
> @package.json 幫我更新依賴說明
💡WSL 無法直接貼圖片　但可用 @ 路徑帶入圖片讓 Claude 分析。

Section 08 · OpenSpec

OpenSpec：Spec-Driven Development

github.com/Fission-AI/OpenSpec

SDD = 規格驅動開發
寫 code 前先和 AI 對「要做什麼」達成共識。spec 是真相來源，code 跟著 spec 走。
/opsx:propose→/opsx:apply→/opsx:archive
terminal-安裝 OpenSpec
$npm i -g @fission-ai/openspec@latest # 安裝 OpenSpec
$openspec init # 專案內初始化


openspec init 後的結構
specs/ — 真相來源
<domain>/spec.md
changes/ — 提案中的修改
<change>/
proposal · design · tasks
config.yaml — 專案設定（可選）
proposal→design→tasks
OpenSpec 採用三階段規格驅動流程：先提出變更提案（Proposal），再設計實作方案（Design），最後拆解執行任務（Tasks）
